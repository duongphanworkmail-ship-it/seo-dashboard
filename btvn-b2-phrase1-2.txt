  ---
  Khi nào chọn Railway

  - Muốn database + app trong cùng 1 nơi (không cần Supabase riêng)
  - App có backend nặng, chạy liên tục (cron job, worker)
  - Thích dashboard quản lý đơn giản, logs rõ ràng

  Khi nào chọn Vercel

  - Project Next.js thuần, ưu tiên tốc độ deploy
  - Muốn zero config, push code là live ngay
  - Cần Edge Functions hoặc CDN toàn cầu tối ưu

  ---
  Khuyến nghị cho bài này

  Nếu dùng Railway: Dùng luôn Railway PostgreSQL thay Supabase — đơn giản hóa
  stack, 1 nơi quản lý tất cả.

  Nếu dùng Vercel: Ghép với Supabase (free tier rộng hơn cho DB).

  Cả hai đều có public URL và đáp ứng được yêu cầu đề bài. Railway phù hợp hơn
  nếu bạn muốn tự quản lý DB và không muốn phụ thuộc nhiều service bên ngoài.

  ✅ Security Coverage Summary

  - NextAuth.js được chọn đúng — có built-in CSRF protection, secure cookies mặc
   định trong production
  - Prisma ORM ngăn SQL injection hoàn toàn ở tầng query
  - Secrets qua env vars — DATABASE_URL, GOOGLE_CLIENT_ID không hardcode
  - requireSession() / requireAdmin() helpers được định nghĩa, intent đúng
  - checkPropertyAccess() có logic phân quyền backend, không chỉ frontend
  - access_type: "offline" + prompt: "consent" đúng để lấy refresh_token

  ---
  ❌ Issues & Risks

  Issue 1: Middleware không bảo vệ API routes

  - Location: middleware.ts, Phase 2 Task 2.9
  - Problem: export { default } from "next-auth/middleware" chỉ guard page
  routes (/dashboard/*, /admin/*). Các route /api/gsc/*, /api/ga4/*,
  /api/admin/* không bị middleware chặn. Plan dựa vào dev nhớ gọi
  requireSession() trong từng route — bỏ sót 1 route là lỗ hổng.
  - Impact: Unauthenticated requests có thể gọi thẳng API routes nếu developer
  quên.

  Fix 1: Tạo API route middleware wrapper

  File: lib/api-handler.ts

  import { getServerSession } from "next-auth"
  import { authOptions } from "@/lib/auth"
  import { NextRequest, NextResponse } from "next/server"
  import type { Role } from "@prisma/client"

  type Handler = (req: NextRequest, session: { user: { id: string; role: Role } 
  }) => Promise<NextResponse>

  export function withAuth(handler: Handler) {
    return async (req: NextRequest) => {
      try {
        const session = await getServerSession(authOptions)
        if (!session?.user) {
          return NextResponse.json({ error: "Unauthorized" }, { status: 401 })
        }
        return await handler(req, session)
      } catch (err) {
        // Never leak internals
        console.error("[API Error]", err)
        return NextResponse.json({ error: "Internal server error" }, { status:
  500 })
      }
    }
  }

  export function withAdmin(handler: Handler) {
    return withAuth(async (req, session) => {
      if (session.user.role !== "ADMIN") {
        return NextResponse.json({ error: "Forbidden" }, { status: 403 })
      }
      return handler(req, session)
    })
  }

  Dùng trong mọi route:

  import { LRUCache } from "lru-cache"

  const rateLimit = new LRUCache<string, number[]>({ max: 500 })

  export function checkRateLimit(identifier: string, maxRequests = 30, windowMs 
  = 60_000): boolean {
    const now = Date.now()
    const timestamps = (rateLimit.get(identifier) ?? []).filter(t => now - t <
  windowMs)
    if (timestamps.length >= maxRequests) return false
    rateLimit.set(identifier, [...timestamps, now])
    return true
  }
